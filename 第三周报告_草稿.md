## 二、策略介绍

本周共实现 **4 个策略**（含第二周的高股息策略作为对照基准）。以下重点介绍本周新增的三个策略及共同改进点。

### 2.1 共同改进：双条件触发 + 智能调仓

第二周高股息策略采用固定频率调仓（月度），无论市场是否变化都会买卖，交易成本偏高。本周在轮动类策略中统一引入两项改进：

| 机制 | 说明 | 目的 |
|:--|:--|:--|
| **双条件触发** | 只有新候选组合的平均涨幅 > 当前持仓平均涨幅 + 阈值（5%）时才调仓 | 避免小幅排名波动导致的频繁换手 |
| **智能调仓** | 调仓时只卖出「不再持有」的股票、只买入「新入选」的股票，已有持仓保留 | 减少交易次数和手续费 |

这两项改进同时应用于 **板块轮动** 和 **个股动量** 两个策略。

---

### 2.2 板块轮动策略

**核心逻辑：**

> 每周检查申万一级行业过去 20 日的平均涨幅，买入涨幅最好的前 3 个行业；每个行业选取沪深300成分股中市值最大的 3 只股票。

**买卖规则：**

| 条件 | 操作 |
|:--|:--|
| 首次建仓 / 新行业组合优势 > 5% | 调仓至目标行业股票 |
| 优势差距不足 5% | 继续持有，不交易 |
| 行业连续入选 | 已有持仓保留，不重复买卖 |

**策略类型：** 趋势型 / 行业轮动

---

### 2.3 个股动量策略

**核心逻辑：**

> 在沪深300成分股中，计算每只股票过去 20 日的涨幅，买入涨幅最大的前 9 只，等权重持有。

**买卖规则：**

| 条件 | 操作 |
|:--|:--|
| 首次建仓 / 新股票组合平均涨幅 > 当前持仓 + 5% | 调仓 |
| 优势差距不足 5% | 继续持有 |
| 股票连续入选 | 已有持仓保留 |

**策略类型：** 趋势型 / 个股动量

**与板块轮动的区别：** 直接在个股层面追涨，不经过行业筛选，进攻性更强，但个股波动也更大。

---

### 2.4 彼得林奇 PEG 策略

**核心逻辑：**

> PEG = PE / 净利润增长率。买入 PEG < 1.0 的低估成长股，持有期间持续监控，直到基本面变贵或趋势反转时卖出。

**买卖规则：**

| 条件 | 操作 |
|:--|:--|
| PEG < 1.0，PE < 40，增长率 > 10% | 买入（沪深300，取 PEG 最低的前 15 只） |
| PEG > 1.5 | 卖出（基本面变贵） |
| 股价从最高点回落 > 15% | 卖出（回撤止盈，锁定利润） |
| 卖出后 3 天内 | 禁止回购同一只股票（冻结机制） |

**策略类型：** 价值型 / 成长价值

**本周新增风控：**

- **回撤止盈（Trailing Stop）** —— 跟踪每只股票的历史最高价，回落超过 15% 自动止盈，避免「坐电梯」
- **卖出冻结** —— 卖出后 3 天内禁止回购，防止震荡行情中反复买卖同一只股票

---

### 2.5 对照基准：高股息/低估值策略（第二周）

第二周已实现的高股息/低估值策略作为本周对照基准，参数不变（全A股选股、20只、月度调仓），用于横向对比不同策略类型的表现差异。

## 三、实现过程与遇到的问题

### 3.1 板块轮动策略

| 问题 | 解决方式 |
|:--|:--|
| 申万行业成分股与沪深300的交集处理 | 先取行业全部成分股，再过滤出属于沪深300的股票；成分股不足 3 只的行业跳过 |
| 行业涨幅计算方式 | 取行业内所有可用成分股的 20 日涨幅算术平均，而非只用龙头 |
| 持仓行业识别 | 缓存 `get_industry()` 结果，避免每周重复查询 |

### 3.2 个股动量策略

| 问题 | 解决方式 |
|:--|:--|
| 动量策略在震荡市频繁换手 | 引入 5% 优势阈值，只有新组合明显更强才换 |
| 调仓时全部清仓再买入，手续费过高 | 改为差量调仓：只买卖变化的部分 |
| 涨幅计算边界 | `get_price` 取 `lookback + 1` 根 K 线，用首尾收盘价计算涨幅 |

### 3.3 彼得林奇 PEG 策略

| 问题 | 解决方式 |
|:--|:--|
| PEG 数据获取 | 通过 `get_fundamentals()` 查询 `valuation.pe_ratio` 和 `indicator.inc_net_profit_year_on_year`，手动计算 PEG |
| 净利润增长率为负或极小导致 PEG 失真 | 过滤 PE > 0 且增长率 > 10% 的股票 |
| 每日检查 vs 回测性能 | 使用 `run_daily(daily_check, time='every_bar')` 每日监控，卖出条件触发时才交易 |
| 回撤止盈的最高价跟踪 | 用全局字典 `g.peak_prices` 记录每只股票的历史最高价，每日更新 |
| 卖出后立即被买回 | 加入 3 天冻结期 `g.frozen_stocks`，防止震荡中反复交易 |

### 3.4 多策略回测对比

| 问题 | 解决方式 |
|:--|:--|
| 不同策略参数不统一，难以公平对比 | 统一基准（沪深300）、初始资金（100万）、手续费（万三）、滑点（0.2%） |
| 策略适用市场不同，单一时间段对比有偏差 | 尽量覆盖较长回测区间（建议 2018–2024），观察牛熊震荡各阶段表现 |

## 五、本周收获与知识补足

### 5.1 新掌握的概念

| 概念 | 理解 |
|:--|:--|
| **行业轮动** | 在行业层面捕捉趋势，比个股动量更分散、波动更小 |
| **动量因子** | 过去 N 日涨幅高的股票短期内可能继续涨（动量效应），但也存在反转风险 |
| **PEG** | 结合估值（PE）和成长（利润增速），比单纯看 PE 更能识别「便宜的成长股」 |
| **回撤止盈** | Trailing Stop，从最高点回落一定比例就卖出，是趋势策略中常用的风控手段 |
| **差量调仓** | 只交易变化的部分，是降低交易成本的有效手段 |
| **优势阈值** | 调仓的「门槛」，避免微小排名变化引发无效交易 |

## 六、改进方向与后续计划

### 6.1 各策略可继续优化的方向

**高股息/低估值：**
- 叠加质量因子（ROE）过滤「假高息」股票
- 行业中性化，避免持仓集中在银行、地产

**板块轮动：**
- 加入大盘择时（如沪深300均线），熊市降低仓位
- 尝试不同回看周期（10日 vs 20日 vs 60日）的参数敏感性测试

**个股动量：**
- 加入波动率过滤，避开高波动「妖股」
- 尝试「动量 + 反转」组合（短期反转 + 中期动量）

**PEG：**
- PEG 计算改用预期增速而非历史增速
- 回撤止盈阈值做参数扫描（10% / 15% / 20%），找到最优值
- 与第二周高股息策略组合，价值 + 成长双引擎
