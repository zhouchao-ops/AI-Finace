## 一、整个过程中遇到的主要问题与解决方式

### 第一周：从零开始的认知门槛

| 问题 | 解决方式 |
|:--|:--|
| 对量化交易只有模糊概念，不知道从何入手 | 先通读第一周任务要求，明确「双均线策略回测」这个具体目标，聚焦学习；不贪多，先跑通一个最简单的策略再说 |
| 聚宽平台的 API 不熟悉，不知道如何获取数据、下单 | 对照聚宽官方文档 + Claude 辅助理解 API 用法，逐行学习示例代码；从 `initialize()` → `run_daily()` → `order_target_value()` 逐步理解每个函数的作用 |
| 回测结果中的指标（夏普比率、最大回撤等）看不太懂含义 | 每遇到一个新概念就用自己的话写下来，并用大白话解释给 AI 听让其确认理解是否正确；比如「夏普比率就是每担一分险能多赚多少钱」 |

**最大教训：** 刚入门时不要追求一次性理解所有概念，而是先跑通一个完整流程，建立"输入→回测→输出"的整体认知，再回头补细节。

### 第二周：从"跑通"到"理解"的跃升

| 问题 | 解决方式 |
|:--|:--|
| 高股息/低估值策略涉及的指标多（股息率、PE、PB），数据来源分散 | 仔细阅读聚宽财报数据 API 文档，区分 `valuation`（市值）和 `indicator`（财务指标）和 `finance`（分红送转）三张表的不同用途；使用 `finance.STK_XR_XD` 表计算实际派息 |
| 调仓频率（月度 vs 季度）对结果影响很大，不知道选哪个 | 两种频率都跑一遍，对比回测结果再做选择；让数据说话，而不是凭感觉判断 |
| 回测效果很好但担心过拟合 | 有意识地将测试区间分成不同市场环境（2018 熊市、2019–2020 反弹、2022 下跌），观察策略在不同阶段的表现是否一致 |

**最大收获：** 回测不是"跑一次看结果"就完事了，需要反复调试参数、对比不同设置，才能真正理解策略的收益来源和风险敞口。

### 第三周：策略横向拓展 + 本地环境搭建

这周实际做了两条平行的探索线：

**线 A — 多策略对比（代码实现层面）：**

| 问题 | 解决方式 |
|:--|:--|
| 板块轮动策略需要申万行业分类，聚宽 API 文档需要查找 | 使用 `get_industries('sw_l1')` 获取全行业列表，`get_industry_stocks()` 获取成分股，结合沪深300做交集筛选 |
| 动量策略在震荡市频繁换手，交易成本高 | 引入「双条件触发」机制——只有新股票组合的平均涨幅超过当前持仓一定阈值（5%）时才调仓；同时实现「差量调仓」——只买卖变化的部分，已有持仓不动 |
| PEG 策略中净利润增长率为负导致 PEG 失真 | 增加过滤条件：只保留 PE > 0 且增长率 > 10% 的股票 |
| 回撤止盈中跟踪最高价，但回测速度慢 | 用 `g.peak_prices` 全局字典记录每只股票的历史最高价，每日在 `daily_check` 中更新 |

**线 B — Qlib 本地环境搭建（系统层面）：**

| 问题 | 解决方式 |
|:--|:--|
| GitHub 下载 cn_data 超大文件超时 | 改用国内镜像 `ghfast.top` 加速下载 |
| MLflow 3.x 与 Qlib 不兼容报错 | 设置 `export MLFLOW_ALLOW_FILE_STORE=true` 绕过兼容性问题 |
| AKShare 批量请求触发限流 | 改用 Baostock 获取前复权日线数据（298/300 只成功），再转为 Qlib 二进制格式 |
| 根分区磁盘空间不足 | 迁移数据到 `/data`（13TB 数据盘） |

**最大体会：** 真正的量化研究不仅写策略代码，数据工程、环境配置、工具链整合占据了很大一部分工作量。"因子→模型→回测→评价"是一条完整的工程流水线。

## 二、如何高效学习和理解新知识

### 2.1 以终为始，任务驱动

每周先看任务要求，明确本周要交付什么（一份报告 + 策略代码/回测结果），然后**倒推**需要学什么。这样不会漫无目的地刷教程，而是带着问题去找答案。实践证明，解决问题的学习效率远高于"先学再用"。

### 2.2 循序渐进，每周一个层次

| 层次 | 对应周 | 学习方式 |
|:--|:--|:--|
| **第一层：跑通流程** | 第一周 | 复制修改他人的完整代码，先让回测跑起来，建立感性认识 |
| **第二层：理解策略** | 第二周 | 自己选策略，从逻辑理解到代码实现独立完成，开始回测优化 |
| **第三层：横向拓展** | 第三周 | 多策略对比，引入新的工具链（Qlib），从规则策略到机器学习策略 |
| **第四层：复盘反思** | 第四周 | 系统性回顾，总结方法论，明确后续方向 |

每一层都建立在前一层的基础上，不跳跃。

### 2.3 三遍学习法

对一个新策略或新概念，我通常过三遍：

1. **第一遍：快速概览** —— 读一遍策略的核心逻辑，不求甚解，只求知道"它在做什么"
2. **第二遍：边做边学** —— 写代码实现或跑回测，遇到不懂的地方再查资料
3. **第三遍：复盘提炼** —— 策略跑完了，回头看哪些地方卡住了，哪些理解了，哪些还是一知半解

### 2.4 用输出倒逼输入

每周写报告本身就是最好的学习方式。写的时候会发现"这个地方其实还没完全搞懂"，然后回去补课。不写出来，很多似是而非的理解就被放过去了。

## 三、AI 工具在学习中起的作用

### 3.1 AI 充当的角色

在这四周的学习中，AI（Claude Code）扮演了多种角色：

| 角色 | 具体作用 | 使用频率 |
|:--|:--|:--|
| **代码助手** | 编写策略代码框架、调试 API 调用错误、优化代码结构 | ⭐⭐⭐⭐⭐ |
| **概念解释器** | 用大白话解释 PEG、夏普比率、IC/IR 等专业概念 | ⭐⭐⭐⭐ |
| **文档导航员** | 帮我在聚宽 API 文档中快速找到需要的函数和参数 | ⭐⭐⭐⭐ |
| **调试伙伴** | 当回测结果异常时，一起分析可能的原因（数据问题、参数问题、逻辑问题） | ⭐⭐⭐ |
| **报告润色** | 帮我整理周报结构和表达，但内容主要由我自己填充 | ⭐⭐⭐ |

### 3.2 具体案例

**案例一：理解 PEG 指标（第二周）**

我最初对 PEG 的理解就停在"PE/增长率"这个公式上。问 AI 后，它不仅解释了公式，还用了一个类比：

> PEG = 0.5 相当于"用 5 块钱买一个每年能赚 10 块钱增长率的公司"，PEG = 2 相当于"用 20 块钱买只赚 10 块钱增长率"，所以 PEG 越低越划算。

这个类比让我立刻理解了为什么 PEG < 1 被认为是低估。

**案例二：调试板块轮动策略（第三周）**

板块轮动策略一开始回测结果特别差，我完全不知道为什么。AI 帮我梳理了排查思路：先确认行业涨幅计算是否正确 → 再确认持仓行业识别是否准确 → 然后看调仓逻辑是否按预期执行。最后发现是 `get_industry_stocks()` 返回的股票和沪深300交集太少，改进了筛选逻辑后结果就正常了。

### 3.3 AI 的局限与注意事项

虽然 AI 帮助很大，但我也体会到它**不能替代独立思考和判断**：

- AI 写代码快，但不会替你理解代码 —— 如果直接把 AI 写的代码贴上去跑，出错了完全不知道怎么修
- AI 对回测结果的"分析"看起来很合理，但可能过拟合历史数据 —— 最终还是需要自己的判断
- AI 有时会推荐过时或错误的 API 用法（例如聚宽某些旧版 API），需要交叉验证

> **我的原则：** AI 是"能对话的 Google + 能写代码的 Stack Overflow"，而不是老师的替代品。理解策略逻辑、分析回测结果、做决定——这些还是得自己来。

## 四、阅读他人分享与交流获得的启发

### 4.1 对"策略一致性的理解"

以前觉得量化就是找到一种圣杯策略，一直用就能稳定赚钱。但看了其他人的实践分享后发现：

- 没有永远有效的策略 —— 适合牛市的策略在熊市可能亏得很惨
- 重要的不是收益最高的策略，而是**自己能理解、能坚持执行的策略**
- **投资组合 ≠ 多个策略简单叠加**，需要考虑策略之间的相关性

### 4.2 对回测的认识

最大的启发来自一句话：

> **"回测的功能是排除错误的策略，而不是发现正确的策略。"**

这意味着：
- 回测结果好 ≠ 实盘一定好（过拟合风险永远存在）
- 回测结果差 ≈ 这个策略大概率不行
- 关键是问自己"为什么这个策略在回测中有效"，如果解释不清楚，大概率是过拟合

### 4.3 量化不只是写代码

群友分享中提到，很多人在量化学习中过分关注写代码和跑回测，而忽略了：

- **对市场的理解** —— 策略到底在赚什么钱？（价值回归？趋势延续？波动率溢价？）
- **风险管理** —— 策略失效时怎么办？最大能承受多少亏损？
- **心态管理** —— 连续回撤时能否坚持执行？

这让我意识到，量化交易终究是**交易**，不是编程比赛。

## 五、对量化交易的阶段性理解与个人心得

### 5.1 量化的本质：用统计规律做决策

四周学习下来，我对量化的理解可以归纳为一句话：

> **量化交易就是用历史数据中的统计规律来指导交易决策，而不是靠直觉或消息。**

这个理解看似简单，但背后有几个重要的含义：

1. **相信概率而不是单次结果** —— 量化不追求每次都猜对，而是追求长期下来期望值为正
2. **纪律性比判断力更重要** —— 最难的往往不是设计策略，而是在回撤期坚持执行策略
3. **可重复、可验证、可改进** —— 这是量化最大的优势，也是和主观交易最本质的区别

### 5.2 四步学习法（个人总结）

经过四周实践，我总结了一套适合自己的量化学习路径：

```
选择方向 → 理解逻辑 → 代码实现 → 回测验证 → 复盘反思
                                                ↓
                                        回到第一步（循环迭代）
```

- **选择方向**：每周一个明确的探索主题（双均线 → 高股息 → 多策略 + Qlib → 复盘）
- **理解逻辑**：先搞懂策略赚什么钱、在什么市场环境下有效、有什么风险，再动手
- **代码实现**：在聚宽/Qlib 上实现策略，跑通回测
- **回测验证**：看收益、看回撤、看夏普、看超额，更重要的是理解为什么有这样的结果
- **复盘反思**：每周写报告，记录踩过的坑和自己的思考

### 5.3 四个策略的横向对比

四周完成了五种策略的调研或实现：

| 策略 | 类型 | 核心逻辑 | 适合市场 | 风控方式 |
|:--|:--|:--|:--|:--|
| 双均线 | 趋势跟踪 | 短线上穿长线买入，下穿卖出 | 强趋势行情 | 均线本身有滞后过滤作用 |
| 高股息/低估值 | 价值型 | 买分红高、PE/PB低的股票 | 震荡市、熊市 | 低估值天然有安全边际 |
| 板块轮动 | 趋势型/行业 | 买入近期涨幅最好的行业 | 牛市、行业分化行情 | 双条件触发减少噪音交易 |
| 个股动量 | 趋势型/个股 | 买入近期涨幅最大的股票 | 牛市 | 优势阈值 + 差量调仓 |
| PEG | 成长价值 | PEG < 1 买入，> 1.5 卖出 | 成长股行情 | 回撤止盈 + 卖出冻结 |

**核心认识：没有完美的策略，只有适合当前市场环境的策略。** 每种策略都有其"甜蜜期"和"失效期"，关键是要知道自己持有的策略在什么情况下会失效。

### 5.4 技术栈演进路径

```
Week 1                  Week 2           Week 3                    Week 4
聚宽(规则策略)  →   聚宽(因子选股)  →   聚宽(多策略) + Qlib(ML)  →   复盘
    │                    │                  │
    ▼                    ▼                  ▼
 双均线回测         高股息/低估值      板块轮动/动量/PEG
                                      Qlib + LightGBM
```

从单平台到多工具、从单策略到多策略、从规则驱动到因子+机器学习，这是一个渐进的过程。

### 5.5 后续方向思考

四周学习摸到了量化交易的"门槛"，但还有很长的路要走。后续可以考虑的方向：

| 方向 | 优先级 | 说明 |
|:--|:--|:--|
| **多因子模型** | ⭐⭐⭐⭐⭐ | 将学到的高股息、动量、PEG 等因子组合成一个多因子打分模型 |
| **Qlib 深入** | ⭐⭐⭐⭐ | 在已有环境基础上，深入理解 Alpha158 因子的计算逻辑，尝试自定义因子 |
| **策略组合** | ⭐⭐⭐⭐ | 将价值型和趋势型策略按权重组合，平衡收益与回撤 |
| **实盘模拟** | ⭐⭐⭐ | 在聚宽模拟盘中验证策略的实盘表现，体验真实市场中的执行问题 |
| **机器学习** | ⭐⭐⭐ | 在 Qlib 中尝试 XGBoost、神经网络等模型，对比 LightGBM 的效果 |

### 5.6 写在最后

四周前，我对量化交易的认知停留在"用电脑自动炒股"这个模糊的印象上。四周后，我至少明白了几个关键问题：

- **量化赚的钱到底从哪来？** —— 从统计规律中寻找市场的非有效性，赚的是定价偏差/趋势延续/风险溢价的回报
- **为什么普通人也能做量化？** —— 聚宽等平台降低了数据获取和回测的技术门槛，开源生态（Qlib、AKShare）让个人研究者也能使用专业工具
- **最难的部分是什么？** —— 不是写代码，而是**思考策略逻辑、理解回测结果、管理自己的预期和心态**

量化交易是一门"越学越觉得自己无知"的领域。四周只是一个开始，后面值得探索的东西还很多。**但这四周建立的方法论——以终为始、输出倒逼输入、循序渐进的层次化学习——不仅适用于量化学习，也适用于其他任何复杂技能的习得。**
